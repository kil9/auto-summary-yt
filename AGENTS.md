# AGENTS.md

## 마인드셋

- 시니어 엔지니어처럼 생각한다.
- 추측으로 뛰어들거나 성급히 결론내리지 않는다.
- 항상 여러 접근을 평가하고, 장점/단점/위험을 각각 한 줄로 적은 뒤 가장 단순한 해법을 선택한다.
- 선택지가 불분명한 경우 사용자에게 선택 가능한 대안을 설명과 함께 제시하고 선택하게 한다.

## 필수 코딩 규칙

- 코딩 전에 Problem 1-Pager: 배경 / 문제 / 목표 / 비목표 / 제약을 작성한다.
- 명시적인 코드를 선호한다; 숨겨진 "매직" 금지.
- DRY를 따르되, 섣부른 추상화는 피한다.
- 구체적으로 예외를 처리하고, 사용자에게 명확한 메시지를 제공한다.
- 관측 가능한 API 동작 변경이 있을 경우 항상 사용자에게 확인할 수 있는 방식을 제공한다.

## 코드 및 파일 참조 규칙

- 코드를 변경하기 전에 정의, 참조, 호출 지점, 관련 테스트, 문서/설정/플래그를 찾아 읽는다.
- 심볼을 수정하기 전에 전역 검색으로 사전/사후 조건을 파악한다.

## 코딩 스타일 및 네이밍 규칙

- 프로젝트의 주 언어/프레임워크 관용 스타일을 따른다.
- 포매터와 import 정리 도구를 우선 적용한다(예: `prettier`).
- 줄 길이는 합리적으로 유지하고, 함수는 짧고 응집력 있게 유지한다.
- 네이밍은 의도를 드러내고, 약어는 팀 컨벤션에서 허용한 경우에만 사용한다.

## 테스트 규칙

- 새 코드에는 새 테스트를 추가한다; 버그 수정에는 회귀 테스트를 반드시 포함한다(먼저 실패하도록 작성).
- 테스트는 결정적이고 독립적이어야 한다.
- E2E 테스트에는 1개 이상의 성공 경로와 1개 이상의 실패 경로를 포함한다.
- 경계값/실패 경로를 기본으로 포함한다.

## 보안 규칙

- 코드/로그/티켓에 비밀값을 절대 남기지 않는다.
- 입력을 검증·정규화·인코딩하고, 파라미터화된 접근을 사용한다.
- 최소 권한 원칙을 적용한다.
- 시크릿/토큰/인증정보는 커밋하지 않는다.
- `host_permissions`는 `youtube.com`과 `gemini.google.com`으로만 제한한다.
- `scripting.executeScript`로 주입하는 코드는 외부 입력을 직접 eval하지 않는다.

## 클린 코드 규칙

- 의도를 드러내는 이름을 사용한다.
- 각 함수는 한 가지 일만 한다.
- 상수는 항상 심볼화한다(하드코딩 금지).
- 코드를 입력 → 처리 → 반환 구조로 구성한다.
- 실패는 구체적인 오류/메시지로 보고한다.

## 안티 패턴 규칙

- 문맥을 읽지 않고 코드를 수정하지 않는다.
- 실패나 경고를 무시하지 않는다.
- 단순히 컨벤셔널하다는 이유로 근거 없는 최적화나 추상화를 도입하지 않는다.
- 광범위한 예외를 남용하지 않는다.

## 커밋 및 풀 리퀘스트 규칙

- 사용자의 명시적인 요청이 있기 전까지는 커밋하지 않는다.
- 커밋을 요청받았을 때는 `AGENTS.md`, `README.md`, `PLAN.md`의 업데이트가 필요한지 검토하고, 필요하면 함께 업데이트한다.
- 기본 브랜치에 직접 푸시하지 않고 PR을 기본 절차로 사용한다.
- 새로운 PR은 항상 Draft로 생성한다.
- 커밋 메시지는 한국어로 작성한다.
- 커밋 메시지 첫 줄에는 80자 이내로 전체 커밋 내용을 요약한다.
- 두 번째 줄은 비워둔다.
- 세 번째 줄부터 목록 형태로 간단하게 변경 내용을 기록한다.

## 이 저장소 전용 정보

### 프로젝트 목적

유튜브 시청 중 버튼 또는 단축키로 Gemini 웹을 분할 화면 또는 백그라운드 탭으로 열고, 현재 유튜브 URL을 Gemini 입력창에 자동 입력·전송하는 크롬 확장. API 사용 없음.

### 핵심 제약

- Gemini API 등 외부 API 사용 금지 — 웹 UI 자동화(DOM 조작) 기반으로만 구현
- Manifest V3 기준 (Service Worker, `chrome.scripting` 등)
- 빌드 도구 없음 — 소스 디렉터리를 직접 크롬에 로드

### 프로젝트 구조

```
manifest.json          # 확장 설정, 권한, 단축키 정의
src/
  background/
    index.js           # Service Worker: 단축키 처리, 탭/창 제어, URL 주입
  content/
    index.js           # Content Script: 유튜브 페이지에 Gemini 버튼 렌더링
  options/
    index.html         # 옵션 페이지 UI
    index.js           # 옵션 페이지 로직 (storage.sync 읽기/쓰기)
    style.css          # 옵션 페이지 스타일
```

### 동작 흐름

1. `src/content/index.js`가 유튜브 영상 페이지에서 제목 옆에 "Gemini" 버튼 삽입
2. 버튼 클릭 → `chrome.runtime.sendMessage({ type: "OPEN_GEMINI", url })` 전송
3. 단축키 `Ctrl+Shift+U` → `chrome.commands.onCommand` 리스너 → 활성 탭 URL 획득
4. `src/background/index.js`의 `openGeminiWithUrl()`이 Gemini 탭/창 생성
   - `openInBackgroundTab: true`: 현재 창에 백그라운드 탭으로 Gemini 열기
   - `openInBackgroundTab: false`: 현재 창을 좌우로 나누고 팝업 창으로 Gemini 열기
5. Gemini 탭 로딩 완료 대기 후 `injectUrlIntoGemini()`로 URL 입력 및 전송 시도

### 설정 옵션 (chrome.storage.sync)

| 키 | 기본값 | 설명 |
|---|---|---|
| `showButton` | `true` | 유튜브 페이지에 Gemini 버튼 표시 여부 |
| `autoInject` | `true` | Gemini 열릴 때 URL 자동 입력 여부 |
| `openInBackgroundTab` | `true` | 백그라운드 탭으로 열기 (false면 분할 창) |

### 빌드/실행/테스트

- **빌드**: 불필요 — 소스 그대로 사용
- **로컬 로드**: `chrome://extensions` → 개발자 모드 → 소스 루트 폴더 선택
- **코드 수정 반영**: 확장 페이지에서 새로고침 아이콘 클릭
- **자동화 테스트**: 없음 (브라우저 환경 의존으로 수동 확인)

### 저장소 특화 보안 체크리스트

- `host_permissions`를 `youtube.com`, `gemini.google.com`으로만 유지
- `scripting.executeScript`에 전달하는 `args`는 URL 문자열만 허용
- DOM 조작 시 `innerHTML` 대신 `textContent` / DOM API 사용
- API 키, 토큰 등 민감 정보를 코드에 하드코딩하지 않음
